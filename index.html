Phase 1: The Clean Slate

START HERE
Do this immediately after installing the Termux app.
Open Termux and grant storage permissions 
termux-setup-storage
(Tap "Allow" on the popup)
Update Core Packages
pkg update -y && pkg upgrade -y
‚ö†Ô∏è CRITICAL: If asked about openssl.cnf or other config files, always type Y and press Enter to accept the package maintainer's version.

Phase 2: The Extraction

Getting your files out of the backup archive.
Copy the Backup from Downloads to Home
cp /sdcard/Download/termux_restore_point_*.tar.gz .
Extract the Archive
tar -xzf termux_restore_point_*.tar.gz
Correct the Folder Structure
The files extract into a subfolder. Move them to the main directory: 
cd termux_backup_*
cp -r * ..
cp .bashrc ..
cp .npmrc ..
cd ..

Phase 3: The "Lazarus" Restoration

Automated installation of software and data.
Run the Restore Script
chmod +x setup-my-grid.sh
./setup-my-grid.sh
Wait for it to finish. It will install Python, Postgres, Redis, Node, and import your database data.
Manual Patch (If script misses it)
If the script finishes but you forgot to add tmux (the grid window manager), install it manually: 
pkg install tmux

Phase 4: The "Handshake" Repairs

CRUCIAL
Security keys (SSH and Cloudflare) break during a reinstall. You must re-link them manually.

üîê A. Re-Link SSH (PC to Phone)

On PC (PowerShell): Remove the old fingerprint
Your PC will block the connection if you don't do this. 
ssh-keygen -R [192.168.100.44]:8022
On Termux: Set a temporary password
passwd
# Type '123456' (blindly) twice
On PC (PowerShell): Push your SSH key to the phone 
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh -p 8022 u0_a728@192.168.100.44 "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys"
On Termux: Restart SSH (Optional but safe) 
pkill sshd && sshd

‚òÅÔ∏è B. Re-Link Cloudflare (The "Ghost Tunnel" Fix)

Your old tunnel credentials are dead. You must create a new tunnel.
Delete the old Ghost Tunnel
# Get the ID of the old tunnel
cloudflared tunnel list
# Delete it
cloudflared tunnel delete <OLD_UUID>
Create a New Tunnel
cloudflared tunnel create redmi-host
üìù COPY THE NEW ID (UUID) PRODUCED BY THIS COMMAND.
Update the Config
nano ~/.cloudflared/config.yml
  • Paste the New UUID into the tunnel: line
  • Paste the New UUID into the credentials-file: path
  • Ensure hostname is just ian-dev.top (NO https://)
Fix DNS (The "Ghost Record" Fix)
  • Go to dash.cloudflare.com ‚Üí DNS
  • Delete the existing CNAME record for ian-dev.top
  • Back in Termux, route the new tunnel:
cloudflared tunnel route dns redmi-host ian-dev.top

Phase 5: Launch

FINAL
Start the Grid
start-server
Verify Everything Works
  • Dashboard should appear (4 panes)
  • Visit https://ian-dev.top ‚Äì App should load

üÜò Troubleshooting Cheat Sheet

psql: FATAL: role "..." does not exist
Fix:
createuser -s $(whoami)
CANNOT LINK EXECUTABLE "node": ... libsqlite3.so
Fix:
ln -s $PREFIX/lib/libsqlite3.so.3 $PREFIX/lib/libsqlite3.so
failed to create tunnel: ... already exists
Fix:
Run this using the UUID from cloudflared tunnel list 
cloudflared tunnel delete <UUID>
DNS record already exists
Fix:
Log into Cloudflare website and manually delete the old CNAME record before running the route dns command
