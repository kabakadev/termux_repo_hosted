Phase 1: The Clean Slate

START HERE
Do this immediately after installing the Termux app.
Open Termux and grant storage permissions 
termux-setup-storage
(Tap "Allow" on the popup)
Update Core Packages
pkg update -y && pkg upgrade -y

Phase 2: The Extraction

Getting your files out of the backup archive.
Copy the Backup from Downloads to Home
cp /sdcard/Download/termux_restore_point_*.tar.gz .
Extract the Archive
tar -xzf termux_restore_point_*.tar.gz
Correct the Folder Structure
The extraction puts files in a folder. We must move them out.
NOTE: We use explicit copies for hidden files (.ssh, .cloudflared) because cp * ignores them. 
cd termux_backup_*
cp -r * ..
cp -r .ssh ..
cp -r .cloudflared ..
cp .bashrc ..
cp .npmrc ..
cd ..

Phase 3: The "Lazarus" Restoration

Automated installation of software and data.
Run the Restore Script
chmod +x setup-my-grid.sh
./setup-my-grid.sh
Wait for it to finish. It will install Python, Postgres, Redis, Node, and import your database data.
Manual Patch (If script misses it)
If the script finishes but you forgot to add tmux (the grid window manager), install it manually: 
pkg install tmux

Phase 4: The "Handshake" Repairs

CRUCIAL
This section has been upgraded to support Dynamic IPs and Username Changes.

üîê A. Re-Link SSH (PC to Phone)

Method 1: ADB (Recommended)
Why this is better: It works even if your WiFi IP changes. It creates a direct tunnel via USB/WiFi Debugging.
1. Get your NEW Username (On Termux) 
whoami
Write down the output (e.g., u0_a245). You CANNOT use your old username.
2. Set a Password (On Termux)
REQUIRED: You cannot push a key via SSH without a password first. 
passwd
# Type '123456' (blindly) twice
3. Restart SSH Server (On Termux) 
pkill sshd
sshd
4. Create the Bridge (PowerShell) 
adb forward tcp:8022 tcp:8022
5. Push Keys (PowerShell)
Replace NEW_USER with the one you found in Step 1. When asked for password, use 123456. 
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh -p 8022 NEW_USER@localhost "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys"
6. Connect
ssh -p 8022 NEW_USER@localhost

‚òÅÔ∏è B. Re-Link Cloudflare (The "Ghost Tunnel" Fix)

Your old tunnel credentials are dead. You must authorize the device, destroy the old tunnel, and create a fresh one from scratch.
1. Authorize Device (Generates cert.pem) 
cloudflared tunnel login
Copy the URL provided into your browser and authorize the domain.
2. Delete the old Ghost Tunnel and Create New 
# Get the ID of the old tunnel
cloudflared tunnel list
# Delete it
cloudflared tunnel delete <OLD_UUID>
# Create new one
cloudflared tunnel create redmi-host
üìù COPY THE NEW ID (UUID) PRODUCED BY THE CREATE COMMAND.
3. Create the Config File
We create the folder first to avoid "No such file" errors, then use Nano for safety. 
mkdir -p ~/.cloudflared
nano ~/.cloudflared/config.yml
Paste the following (Replace PUT_UUID_HERE with your new ID): 
tunnel: PUT_UUID_HERE
credentials-file: /data/data/com.termux/files/home/.cloudflared/PUT_UUID_HERE.json

ingress:
  - hostname: ian-dev.top
    service: http://localhost:5000
  - service: http_status:404
4. Fix DNS (The "Ghost Record" Fix)
  • Go to dash.cloudflare.com ‚Üí DNS
  • Delete the existing CNAME record for ian-dev.top
  • Back in Termux, route the new tunnel:
cloudflared tunnel route dns redmi-host ian-dev.top

üÜò Troubleshooting Cheat Sheet

Connection closed by 127.0.0.1 port 8022
Fix:
This means you are using the wrong username (from an old install) OR sshd is not running. Run whoami on phone, pkill sshd && sshd, and update your Windows ssh command.
ssh: connect to host 192.168.100.x port 8022: Connection refused
Fix:
Your IP changed or ADB is disconnected.
adb forward tcp:8022 tcp:8022
ssh -p 8022 NEW_USER@localhost
u0_aXXX@localhost: Permission denied (publickey,password)
Fix:
You haven't set a password on Termux yet. Run passwd on the phone, set it to 123456, then try the Windows command again.
bash: .../.cloudflared/config.yml: No such file or directory
Fix:
The folder does not exist. Run mkdir -p ~/.cloudflared before trying to create the file.
